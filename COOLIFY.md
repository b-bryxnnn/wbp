# คู่มือการตั้งค่า Coolify สำหรับโปรเจกต์ wbp-vote-app

**ตอบคำถาม:** "การตั้งค่าพอร์ตใน coolify สามารถเลือกได้เองเลยใช่มั้ย"
**คำตอบ:** ได้ครับ! คุณสามารถเลือกพอร์ต (Port) ที่ต้องการเปิดใช้งานได้อิสระ เพื่อไม่ให้ชนกับแอปอื่นในเครื่อง Home Server ของคุณ

วิธีเปลี่ยนพอร์ตทำได้ 2 วิธีง่ายๆ:

### วิธีที่ 1: ตั้งค่าผ่าน Environment Variables (แนะนำ)
ผมได้เตรียมตัวแปรไว้ใน `docker-compose.yml` ให้แล้ว คุณสามารถกำหนดค่าเหล่านี้ใน Coolify ได้เลย:

| ชื่อตัวแปร (Key) | ค่าตัวอย่าง (Value) | คำอธิบาย |
| :--- | :--- | :--- |
| **`APP_PORT`** | `3001` | เปลี่ยนพอร์ตเว็บแอป (Default: 3000) |
| **`ADMINER_PORT`** | `8081` | เปลี่ยนพอร์ต Adminer (Default: 8080) |
| **`DB_PORT`** | `5433` | เปลี่ยนพอร์ต Database (Default: 5432) |

### วิธีที่ 2: แก้ไขไฟล์ Docker Compose โดยตรง
ในหน้า Configuration ของ Stack ใน Coolify คุณจะเห็นโค้ด `docker-compose.yml` สามารถแก้ตัวเลขหน้าเครื่องหมาย `:` ได้เลยครับ
เช่น เปลี่ยน `"3000:3000"` เป็น `"8888:3000"` (เข้าผ่านพอร์ต 8888 แทน)

---

## สรุปขั้นตอนใน Coolify แบบเข้าใจง่าย

วิธีนี้ Coolify จะสร้างทั้งเว็บ, ฐานข้อมูล, และ Adminer ให้พร้อมกัน และเชื่อมต่อกันเองอัตโนมัติครับ

1.  **ใน Coolify:** สร้าง Resource ใหม่ เลือก **Docker Compose (Stack)**
2.  **แหล่งข้อมูล (Source):** เลือกเชื่อมต่อกับ **GitHub Repository** ของโปรเจกต์นี้
3.  **Configuration:** Coolify จะอ่านไฟล์ `docker-compose.yml` ในโปรเจกต์
4.  **กด Deploy ได้เลย!**

*(ระบบจะใช้ค่าพื้นฐานใน `docker-compose.yml` ทำให้เว็บเชื่อมต่อกับฐานข้อมูลได้ทันทีโดยไม่ต้องตั้งค่า Environment Variables เพิ่มในครั้งแรก)*

---

## การปรับแต่งความปลอดภัย (Production)

เมื่อ Deploy ผ่าน Stack จนใช้งานได้แล้ว แนะนำให้มาแก้ค่าเหล่านี้เพื่อความปลอดภัยครับ:

1.  ไปที่เมนู **Environment Variables** ของ Stack หรือ Service `app`
2.  เพิ่มค่าดังนี้:

| ชื่อตัวแปร (Key) | ค่าตัวอย่าง (Value) |
| :--- | :--- |
| **`JWT_SECRET`** | ตั้งรหัสยาวๆ เช่น `my-super-secret-key-1234` (ระบบ Login จะปลอดภัยขึ้น) |
| **`DATABASE_URL`** | (ถ้าต้องการเปลี่ยนรหัส Database ค่อยแก้ตรงนี้) |

---

## คำสั่งสำคัญหลัง Deploy (ห้ามลืม)

ไม่ว่าจะใช้วิธีไหน หลังจาก Deploy ครั้งแรกเสร็จ ต้องสั่งให้ระบบสร้างตารางฐานข้อมูลครับ:

1.  ไปที่ Service **app** (หรือ wbp-nextjs) ใน Coolify
2.  เข้าเมนู **Terminal** หรือ **Execute Command**
3.  รันคำสั่ง:
    ```bash
    npx prisma migrate deploy
    ```
4.  (แนะนำ) ลงข้อมูลตัวอย่าง (โรงเรียน/User):
    ```bash
    npm run prisma:seed
    ```
